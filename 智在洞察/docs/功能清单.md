## 三、技术架构要点（Go语言）

### 3.1 核心技术栈

| 层级 | 技术选型 | 说明 |
|-----|---------|------|
| Web框架 | Gin / Fiber | 高性能HTTP框架 |
| ORM | GORM / Ent | 数据库ORM |
| 缓存 | go-redis | Redis客户端 |
| 消息队列 | sarama(Kafka) / amqp(RabbitMQ) | 异步消息 |
| 配置管理 | Viper | 多格式配置支持 |
| 日志 | Zap / Logrus | 结构化日志 |
| 定时任务 | robfig/cron | 分布式定时任务 |
| 权限 | Casbin | RBAC权限框架 |
| 对象存储 | MinIO SDK | 文件存储 |
| 搜索引擎 | go-elasticsearch | 全文检索 |
| 服务发现 | Consul / Etcd | 微服务注册发现 |
| 链路追踪 | OpenTelemetry | 分布式追踪 |

### 3.2 关键设计模式

1. **管道模式(Pipeline)**: 数据采集、清洗、存储流程
2. **工作池模式(Worker Pool)**: 并发任务处理
3. **发布订阅模式(Pub/Sub)**: 消息推送、事件通知
4. **策略模式(Strategy)**: 解析规则、风控规则切换
5. **模板方法模式(Template)**: 报告生成流程

### 3.3 性能优化要点

1. **并发控制**: goroutine池化，避免无限制创建
2. **缓存策略**: 多级缓存，热点数据预加载
3. **数据库优化**: 读写分离、分库分表、索引优化
4. **异步处理**: 耗时操作异步化，提升响应速度
5. **连接池**: 数据库、Redis、HTTP连接复用

## 六、附录

### 6.1 模块依赖关系

```
基础架构与底座 → 所有模块
用户体系管理 → 所有需要权限控制的模块
嵌套附件穿透式解析 → 场景编排管理、报告生成
场景编排管理 → 报告集市、AIGC作战支撑
```

### 6.2 数据流架构

```
外部数据源 → 爬虫采集 → 数据清洗 → 结构化存储
                                    ↓
用户请求 → 场景编排 → 报告生成 → 报告集市
                    ↓
                AI分析 → 智能推送 → 用户端
```
